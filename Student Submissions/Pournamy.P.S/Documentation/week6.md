During this week, we focused on understanding **DAX (Data Analysis Expressions)** and its role in Power BI. DAX is used to create calculations, measures, and derived columns that help in performing advanced data analysis. It combines concepts from Excel formulas and SQL, making it a powerful tool for handling complex computations such as aggregations, filtering, and data transformations. It is often referred to as the “brain” of Power BI because it enables deeper insights beyond basic visualizations.

We also explored the fundamentals of effective dashboard design. A well-structured dashboard is built to answer key analytical questions and present insights clearly. Important elements of a dashboard include a clear and meaningful title, KPI cards, and slicers. The title should be simple and descriptive so that users can quickly understand the purpose of the dashboard.

KPI cards are used to highlight key metrics and are typically the first elements noticed by users. These should be designed carefully to represent the most important values in the analysis. All dashboard elements should be interconnected, meaning that user interactions such as filtering or selecting data in one visual will dynamically update other visuals on the dashboard.

Slicers are used to filter data interactively, allowing users to focus on specific aspects such as time, location, or conditions. Additionally, when multiple dashboard pages are created, they should be consistent and logically connected so that users can easily navigate between them and maintain a clear understanding of the analysis.
