This week focused on understanding fundamental concepts of data modeling and their practical application in Power BI. The primary topic covered was the structure and role of fact tables and dimension tables, where the fact table acts as the central component storing measurable data and key references, while dimension tables provide detailed descriptive information. These components together form a star schema, which is widely used for organizing data efficiently for analysis. Additionally, different types of relationships between tables were studied, including one-to-one, one-to-many, and many-to-many, collectively referred to as cardinality in data modeling.

Further, the process of importing data into Power BI was explored, particularly from Excel files. It was observed that importing creates a static copy of the data, meaning any updates made in the source file are not automatically reflected, making this approach less suitable for dynamic data scenarios. To address this limitation, the concept of connecting Power BI directly to databases was introduced, where data updates in the database are reflected in the reports. This connection is enabled through APIs, which act as a bridge by providing a set of rules and standards for communication between systems. The concept of live data modeling was also discussed, highlighting its importance in enabling real-time or near real-time data updates, although certain limitations such as refresh constraints were noted.
